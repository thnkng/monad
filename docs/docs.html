Design System

Manifest

A minimalist atomic design system built on principles of clarity, visual silence, and emotional precision.

What is Monad?

Monad is a complete design system for building modern interfaces with semantic HTML at its core. Every component is crafted to disappear ‚Äî to serve human experience without noise.

Born from frustration with bloated frameworks and decorative design, Monad strips everything to its essence. What remains is pure, intentional, and beautiful.

Architecture

Monad follows the Atomic Design methodology with a semantic-first approach:

Tokens

Foundation layer. Colors, typography, spacing, motion, effects. Every value is a token ‚Äî no hard-coded pixels.

Base

HTML resets and semantic typography. Clean slate for consistent rendering across browsers.

Atoms

HTML tags styled directly: <button>, <input>, <nav>, <header>, <section>. No wrapper divs required.

Molecules

Simple components with single responsibility: .card, .badge, .avatar, .dropdown. CSS classes for composite elements.

Organisms

Complex components using semantic tags: <aside.sidebar>, <nav.breadcrumb>. Structure with meaning.

Templates

Page layouts as body variants: <body.app-layout>, <body.dashboard-layout>, <body.auth-layout>. Complete layout systems.

Utilities

Layout helpers: margin, padding, grid, flexbox. When semantic HTML isn't enough.

Technology Stack

Pure SCSS ‚Äî No preprocessor complexity. Clean, readable, maintainable.

Web Components ‚Äî Only for interactive elements (modal, dropdown, tabs, toast). Encapsulated, reusable, framework-agnostic.

Zero Dependencies ‚Äî Except Lucide icons for visual consistency. No jQuery. No Bootstrap. No bloat.

Semantic HTML ‚Äî Accessibility built-in. SEO-friendly. Standards-compliant.

  • Clarity before form ‚Äî Every element exists for a reason
  • Visual silence ‚Äî Space is as important as content
  • Emotional precision ‚Äî Colors and names evoke feeling
  • Scalability without noise ‚Äî Grow without losing harmony

Philosophy

Monad is born from the idea that design should not merely decorate technology ‚Äî it must reveal its essence. Every detail is an invitation to clarity, coherence, and visual silence.

The Five Principles

These principles guide every decision in Monad ‚Äî from naming conventions to color choices, from spacing tokens to interaction patterns.

1. Clarity before form

Every element must exist for a reason. No pixel is decorative; everything communicates. Beauty lies in precision, not ornamentation.

In practice: Semantic HTML tags styled directly. Minimal class dependencies. Every line of code serves a purpose.

2. Visual silence

Space is as important as content. Elements are silent by default ‚Äî interactions require opt-in. Generous spacing allows the eye to breathe.

In practice: .hoverable class required for hover effects. Never automatic. Silence is the default state.

3. Emotional precision

Interfaces are never neutral ‚Äî they evoke trust, lightness, calm. Every curve, color, and rhythm carries emotional intent. Naming is poetic.

In practice: Colors named graphite-mind, cloud-silent, ember-energy. Motion curves evoke lightness or urgency.

4. Scalability without noise

A system is only good if it can grow without losing harmony. We seek universal patterns, not fleeting conventions. Auto-generate variants via loops.

In practice: SCSS loops generate all color variants automatically. No hard-coding. Tokens power everything.

5. Humanity at the center

Invisible technology ‚Äî disappears so human experience prevails. In the end, everything is about people ‚Äî not code. Developer experience (DX) is user experience.

In practice: Clean API. Semantic naming. Comprehensive accessibility. Built for humans first, machines second.

Design Reveals, Not Decorates

Modern interfaces are drowning in noise. Shadows stacked on shadows. Colors fighting for attention. Animations that distract rather than guide.

Monad takes a different path. We embrace silence. We give space to breathe. We design with intention, not trends.

Every shadow has a reason. Every color communicates. Every animation guides the eye. Nothing is decoration ‚Äî everything is communication.

Technology That Disappears

The best interface is one you don't notice. The best code is one that feels inevitable. The best design system is one that gets out of the way.

Monad is built to disappear. Pure SCSS. No frameworks. No dependencies beyond Lucide icons. Semantic HTML at its core. Web Components only when necessary.

Technology should serve humans, not the other way around. Monad serves.

"Design reveals, not decorates."

Spacing

Base unit: 4px. Scale: --space-1 (4px) to --space-8 (40px).

Typography

Font family: Inter. Sizes from --font-size-xsmall (12px) to --font-size-5xlarge (48px). Weights: 300-700.

Motion

Durations: fast (150ms), medium (250ms), slow (400ms). Easing: standard, decelerate, accelerate.

Effects

Radius: --radius-sm/md/lg/pill. Shadows: --shadow-faint/soft/medium. Focus ring: 2px with 40% opacity.

Base Styles Semantic HTML

Foundation layer that resets browser inconsistencies and establishes semantic HTML typography. No classes required ‚Äî just write HTML.

Philosophy

Base styles make HTML beautiful by default. Every semantic tag receives intentional styling. Headers, paragraphs, lists, code blocks ‚Äî all work out of the box.

This is where "semantic-first" becomes real. Write <h1>, not <div class="title">. Write <blockquote>, not <div class="quote">.

Organisms Complex Patterns

Complex components composed of molecules and atoms using semantic HTML. Organisms define complete interface patterns like sidebars, navigation systems, and empty states.

Philosophy

Organisms are where design systems come alive. They combine atoms and molecules into complete, functional interface sections. In Monad, organisms use semantic HTML tags (<aside>, <nav>, <header>) as their foundation, not generic divs.

Technical Implementation

File Structure

package/styles/themes/
‚îú‚îÄ‚îÄ _index.scss          # Theme exports
‚îú‚îÄ‚îÄ _moonlight.scss      # Light theme colors
‚îî‚îÄ‚îÄ _midnight.scss       # Dark theme colors

How It Works

  1. CSS Custom Properties: All colors defined as --variable-name in theme files
  2. Selector Specificity: body[theme="midnight"] overrides :root defaults
  3. Token References: tokens/_colors.scss now references var(--color-name)
  4. Component Agnostic: All components automatically adapt via CSS custom properties
  5. No JavaScript: Pure CSS solution ‚Äî change theme by modifying HTML attribute

Best Practices

‚úÖ Do

  • Use Web Components for interactions requiring JavaScript
  • Combine with CSS classes for styling (e.g., .hoverable)
  • Listen to custom events for integration with your app
  • Use semantic HTML inside slots (proper headings, buttons)
  • Test keyboard navigation and screen readers

‚ùå Don't

  • Don't use Web Components for static content (use CSS molecules)
  • Don't override internal Shadow DOM styles (use CSS parts if needed)
  • Don't nest modals or tooltips inside each other
  • Don't forget to clean up event listeners in SPA contexts
  • Don't rely on JavaScript when CSS can achieve the same result

Performance Notes

  • Lazy Loading: Components register only when first used
  • Intersection Observer: ActiveNav uses efficient scroll tracking
  • Event Delegation: Minimal event listeners per component
  • No Dependencies: Pure vanilla JavaScript, no framework overhead
  • Tree-Shakeable: Import only components you need

Framework Integration

Monad Web Components work in any framework:

  • React: Use ref to access component methods
  • Vue: Use @open syntax for events
  • Angular: Import in CUSTOM_ELEMENTS_SCHEMA
  • Svelte: Use bind:this for references
  • Vanilla: Direct DOM API usage
