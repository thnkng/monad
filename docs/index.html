Monad

Sistema de design minimalista que revela a essência da tecnologia através da clareza, do silêncio visual e da precisão emocional.

Manifesto

O Monad nasce da ideia de que o design não deve apenas decorar a tecnologia — ele deve revelar sua essência. Cada detalhe é um convite à clareza, à coerência e ao silêncio visual.

Clareza antes da forma

Cada elemento deve existir por um motivo. Nenhum pixel é decorativo; tudo comunica. A beleza está na precisão.

Silêncio visual

O espaço é tão importante quanto o conteúdo. O olhar precisa de respiro para entender, e o usuário, de tempo para sentir.

Precisão emocional

Interfaces não são neutras. Elas provocam confiança, leveza, calma. Cada curva, cor e ritmo carrega uma intenção emocional.

Escalabilidade sem ruído

Um sistema só é bom se puder crescer sem perder sua harmonia. O Monad busca padrões universais, não convenções passageiras.

Humanidade no centro

Tecnologia invisível é aquela que desaparece para que a experiência humana prevaleça. No fim, tudo é sobre pessoas — não sobre código.

Sobre o Monad

Monad é um sistema de design atômico construído sobre princípios de clareza e minimalismo. Cada componente é projetado para ser inevitavelmente simples.

Princípios

  • Design Atômico: tokens → átomos → moléculas → organismos
  • Progressivo: funciona sem JavaScript, melhora com ele
  • Escalável: padrões compostos, não classes complexas
  • Acessível: semântica HTML, contraste adequado, navegação por teclado

Composição

O Monad usa primitivas de layout para compor interfaces consistentes

Primitivas de Layout

Classe Uso Exemplo
container Limita largura máxima (1152px) e centraliza Seções principais
container-narrow Largura máxima reduzida (768px) para leitura Artigos, textos longos
stack-* Espaçamento vertical entre elementos Listas, formulários
cluster-* Espaçamento horizontal com wrap Botões, tags
grid Layout em grade responsiva Cards, galerias
flex-* Alinhamento flexbox Headers, toolbars

Estados Interativos

Princípio fundamental: Silêncio por padrão — interatividade é sempre opt-in

Utilitários de Hover

Adicione a classe .hoverable combinada com variantes para criar efeitos ao passar o mouse:

Classe Efeito Uso
.hoverable.hoverable-lift Eleva o elemento com sombra Cards, botões principais
.hoverable.hoverable-scale Aumenta levemente o tamanho Imagens, ícones
.hoverable.hoverable-glow Adiciona sombra brilhante CTAs, destaques
.hoverable.hoverable-fade Reduz opacidade Links secundários
.hoverable.hoverable-brighten Aumenta brilho Imagens, fundos escuros 
<div class="card hoverable hoverable-lift">
  Card com efeito de elevação ao hover
</div>

<button class="graphite-core hoverable">
  Botão com hover básico
</button>

<img src="..." class="hoverable hoverable-scale" alt="...">

Utilitários de Pressão (Click)

Use .pressable para feedback visual ao clicar:

Classe Efeito Uso
.pressable Redução de escala sutil (0.98) Botões, cards clicáveis
.pressable.pressable-lift Move para baixo (translateY) Botões físicos, switches
.pressable.pressable-shrink Redução mais pronunciada (0.95) Elementos grandes, destaque forte 
<div class="card pressable" onclick="...">
  Card clicável com feedback
</div>

<button class="graphite-core pressable pressable-lift">
  Botão com efeito de pressão
</button>

Utilitário Combinado: .interactive

Combina hover + press para elementos interativos completos:

<div class="card interactive">
  Card com hover (lift) e press (scale) automáticos
</div>

<!-- Equivalente a: -->
<div class="card hoverable hoverable-lift pressable">
  Mesmo comportamento, mais verboso
</div>
Hoverable Lift

Átomos vs Moléculas

Átomos

Use quando precisar apenas de elementos HTML básicos:

  • Formulários simples
  • Tabelas de dados
  • Botões individuais
  • Links de navegação
  • Imagens e vídeos

Vantagem: Sem classes necessárias, HTML semântico puro

Moléculas

Use quando precisar de componentes compostos:

  • Cards de conteúdo
  • Alertas e notificações
  • Modais e overlays
  • Dropdowns e menus
  • Avatares com badge

Vantagem: Componentes complexos prontos para uso

Sistema de Cores

Hierarquia Semântica

Paleta Propósito Quando Usar
Graphite Paleta primária Ações principais, navegação, hierarquia visual
Thought Paleta secundária Ações secundárias, elementos complementares
Flow Estado de sucesso Confirmações, conclusões bem-sucedidas
Pulse Estado de aviso Alertas, atenção necessária, mudanças pendentes
Break Estado de erro Erros, ações destrutivas, validações falhas
Cloud/Silver Cores neutras Fundos, bordas, texto secundário

Tons e Aplicações

Graphite (4 tons)

mind (mais escuro) → deep → core → light (mais claro)

Thought (3 tons)

default → focus → depth

States (tom único)

flow, pulse, break — cores saturadas para feedback imediato

Boas Práticas

✅ Faça

  • Use tokens de espaçamento (--space-*)
  • Adicione .hoverable apenas onde faz sentido
  • Combine primitivas de layout (stack + cluster)
  • Use nomes de tamanho consistentes: small, medium, large, extra-large
  • Prefira HTML semântico (átomos) quando possível
  • Use .interactive para elementos clicáveis óbvios
  • Teste contraste de cores (mínimo 4.5:1 para texto)

❌ Evite

  • Hard-coded pixels (padding: 16px)
  • Hover automático em tudo (viola silêncio visual)
  • Abreviações inconsistentes (sm, md, lg)
  • Classes repetidas (avatar avatar-large)
  • Elementos decorativos sem propósito semântico
  • Múltiplas variantes de hover no mesmo elemento
  • !important fora de utilitários

Acessibilidade

Checklist Essencial

  • Contraste: Todas as cores passam WCAG AA (4.5:1 para texto normal, 3:1 para texto grande)
  • Foco visível: Estados :focus-visible em todos os elementos interativos
  • Navegação por teclado: Tab, Enter, Escape funcionam em modais e dropdowns
  • ARIA: Labels adequados em controles de formulário e componentes dinâmicos
  • Semântica HTML: Botões são <button>, links são <a>
  • Mensagens de erro: Associadas com aria-describedby
  • Animações: Respeitam prefers-reduced-motion

Exemplo de Implementação Acessível

<!-- Formulário acessível -->
<form class="stack-compact">
  <label for="email">Email</label>
  <input 
    type="email" 
    id="email" 
    aria-describedby="email-error"
    aria-invalid="true"
  >
  <span id="email-error" class="text-break">
    Email inválido
  </span>
  
  <button type="submit" class="graphite-core hoverable">
    Enviar
  </button>
</form>

<!-- Modal acessível (Web Component) -->
<monad-modal 
  id="meu-modal" 
  aria-labelledby="modal-title"
  role="dialog"
>
  <h3 id="modal-title">Título do Modal</h3>
  <p>Conteúdo...</p>
  <button onclick="this.closest('monad-modal').close()">
    Fechar
  </button>
</monad-modal>

Instalação e Uso

1. Incluir CSS

<link rel="stylesheet" href="dist/styles/monad.css">

2. Incluir JavaScript (Opcional)

Necessário apenas para Web Components (Modal, Dropdown, Alert)

<script src="dist/monad.js" type="module" defer></script>

3. Estrutura Básica

<!DOCTYPE html>
<html lang="pt-BR">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Meu App</title>
  <link rel="stylesheet" href="dist/styles/monad.css">
  <script src="dist/monad.js" type="module" defer></script>
</head>
<body>
  <header class="graphite-mind">
    <div class="container">
      <h1>Meu App</h1>
    </div>
  </header>

  <main>
    <section class="section-spacious">
      <div class="container">
        <div class="stack-base">
          <!-- Seu conteúdo aqui -->
        </div>
      </div>
    </section>
  </main>
</body>
</html>
