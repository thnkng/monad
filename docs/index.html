Monad

Sistema de design minimalista que revela a essência da tecnologia através da clareza, do silêncio visual e da precisão emocional.

Manifesto

O Monad nasce da ideia de que o design não deve apenas decorar a tecnologia — ele deve revelar sua essência. Cada detalhe é um convite à clareza, à coerência e ao silêncio visual.

Clareza antes da forma

Cada elemento deve existir por um motivo. Nenhum pixel é decorativo; tudo comunica. A beleza está na precisão.

Silêncio visual

O espaço é tão importante quanto o conteúdo. O olhar precisa de respiro para entender, e o usuário, de tempo para sentir.

Precisão emocional

Interfaces não são neutras. Elas provocam confiança, leveza, calma. Cada curva, cor e ritmo carrega uma intenção emocional.

Escalabilidade sem ruído

Um sistema só é bom se puder crescer sem perder sua harmonia. O Monad busca padrões universais, não convenções passageiras.

Humanidade no centro

Tecnologia invisível é aquela que desaparece para que a experiência humana prevaleça. No fim, tudo é sobre pessoas — não sobre código.

Sobre o Monad

Monad é um sistema de design atômico construído sobre princípios de clareza e minimalismo. Cada componente é projetado para ser inevitavelmente simples.

Princípios

  • Design Atômico: tokens → átomos → moléculas → organismos
  • Progressivo: funciona sem JavaScript, melhora com ele
  • Escalável: padrões compostos, não classes complexas
  • Acessível: semântica HTML, contraste adequado, navegação por teclado

Diretrizes

Princípios de composição e uso do sistema

Composição

O Monad usa primitivas de layout para compor interfaces consistentes

Primitivas de Layout

Classe Uso Exemplo
container Limita largura máxima (1152px) e centraliza Seções principais
stack-* Espaçamento vertical entre elementos Listas, formulários
cluster-* Espaçamento horizontal com wrap Botões, tags
grid Layout em grade responsiva Cards, galerias
flex-* Alinhamento flexbox Headers, toolbars

Átomos vs Moléculas

Átomos

Use quando precisar apenas de elementos HTML básicos:

  • Formulários simples
  • Tabelas de dados
  • Botões individuais
  • Links de navegação
  • Imagens e vídeos

Moléculas

Use quando precisar de componentes compostos:

  • Cards de conteúdo
  • Alertas e notificações
  • Modais e overlays
  • Dropdowns e menus
  • Avatares com badge

Sistema de Cores

Hierarquia Semântica

  • Graphite (Primary): Ações principais, navegação primária
  • Thought (Secondary): Ações secundárias, complementos
  • Flow (Success): Confirmações, estados positivos
  • Pulse (Warning): Avisos, atenção necessária
  • Break (Error): Erros, ações destrutivas
